# Backlog

用于记录暂时不做、后续再做、或者等待拆解的事项。

## 高优先级

- 重启程序并验证 `FetchTransferLogs` 的多日志入库结果
- 使用现有 API 验证同一个地址是否能查到多条转账记录
- 根据运行结果决定是否调整区块范围
- 为 `GET /transfers?address=...` 增加更多测试地址样例

## 中优先级

- 增加数据库初始化脚本
- 增加示例请求和响应样例
- 增加最小架构图的可视化版本
- 将区块范围、HTTP 端口等配置从代码常量逐步移到环境变量
- 考虑为 `transfer_events` 添加 `block_hash` 或 `indexed_at`

## 低优先级

- 评估 WebSocket 实时监听的时机
- 评估 Rust 重写 Indexer 的时机
- 规划阶段 2 的地址分析维度
- 规划阶段 3 的基础风控规则
- 规划阶段 4 的 LLM 接入方式

## 明确暂时不做

- 多链支持
- 跨链桥风险分析
- AI Agent 编排
- RAG
- 复杂部署和云原生体系
- 高级性能优化
- 生产级 indexer 断点续扫
- 大规模并发抓取

## 后续维护规则

每次协作后，优先检查：

- 哪些事项已经完成，可以移除或标记
- 哪些事项变成了当前阻塞点
- 哪些新任务应该加入 backlog
